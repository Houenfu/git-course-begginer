git --version:查看git版本
git config --global --list:查看git的user
git config --global --user.name "xxxx":重命名user.name为xxxx
git config --global --user.email "xxxx@fake.com":为user设置邮箱
git status:查看git状态
git init:在个人项目中创建一个本地仓库！.git的一个文件夹
git init -b main:是一个Git命令，用来初始化一个新的仓库并指定默认分支名为main
git add <file>:将file放入到暂存区
git rm --cached <file>:将file从暂存区移除
git commit -m "My first commit":把已经添加到暂存区的文件保存为一次提交，并附上提交说明"My first commit"
git log:用来查看仓库里的提交历史
git diff:用来查看文件的改动内容, 比较的是工作区与暂存区的差异
git diff --staged:是用来查看暂存区与仓库之间的差异
git add .:将目录的所有文件全部添加至暂存区！
git rm --cached creads.txt:删除文件并将文件添加至暂存区，然后手动commit提交改动，这样才能删除这个文件！
git colone http://...:是Git用来从远程仓库克隆（下载）整个项目到本地的命令！
mkdir git-course:创建名为git-course的文件夹
cd git-course:跳转到git-course
dir:列出当前目录下的文件或文件夹
ls:列出当前目录下的文件或文件夹（按列排序，不显示隐藏文件），跟windows中的dir差不多！
echo "# git-course demo" >> README.md:把文本"# git-course demo"添加到README.md文件末尾
cat <file>:用来查看文件内容！
git branch -M main:重命名当前的分支为main，并强制覆盖已存在的分支（如果有的话）
ssh-keygen -o:生成一个新的SSH密钥对，其中-o参数指定使用OpenSSH新的密钥格式，而不是旧的PEM格式
ssh-keygen -o-->cd C:\users\32318-->cd .ssh-->ls-->cat xxxx.pub:获取ssh密钥，并且添加到GitHub
git remote add origin git@github.com:Houenfu/git-course-begginer.git:添加远程仓库并起名为origin
git push -u origin main:把本地main分支的提交推送到远程仓库origin，设置上游
pwd:输出当前所在目录
git push origin main:把本地main分支提交推送到原曾仓库origin，不设置上游
git tag -a v1.0 -m "1st release":创建一个带注释的标签
git show v1.0:查看标签v1.0指向的commit详情
git tag v1.1 -m "29th Sep release":创建一个标签v1.1并添加说明
git push origin v1.1:将本地标签v1.1推送到远程仓库
git tag:列出本地仓库中已有的所有标签（tags）
git clone git@github.com:Houenfu/git-course-begginer.git:通过SSH，git clone到本地！前提是需要ssh密钥
git log --pretty=oneline:用一行显示每个提交（commit）记录，比默认的git log简洁得多
git remote -v:查看本地仓库关联的远程仓库的地址
git cheackout -b feature1:在当前分支的基础上创建一个新分支feature1
git branch:列出本地所有分支，当前所在分支会在名字前用*标记
git switch feacher1:切换到已有分支feacher1
git branch -c feacher2:创建并切换新分支feacher2
git branch --all:列出本地和远程所有分支，本地分支会直接显示，远程分支会带上远程仓库名称前缀
git switch -:回到上一个分支
git branch -d feacher3:删除本地分支feacher3
